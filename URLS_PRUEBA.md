## ⚠️ Notas Importantes

### URLs que Funcionan Bien
- Artículos de noticias estándar
- Blogs técnicos
- Páginas con contenido claro

### URLs que Pueden Fallar
- Páginas con mucho JavaScript
- Sitios con paywall
- Páginas que requieren login
- URLs muy cortas o sin contenido

### Recomendaciones
1. Probar las URLs antes de grabar
2. Tener 5-6 URLs de respaldo
3. Usar artículos recientes
4. Preferir sitios conocidos
5. Evitar URLs muy largas

## 🎥 Script del Video

### Minuto 0-1: Introducción
"Hoy voy a demostrar mi proyecto de Article Extractor & Summarizer, un sistema completo con CRUD que usa inteligencia artificial para resumir artículos web."

### Minuto 1-2: Arquitectura
"El proyecto usa React + TypeScript en el frontend, Spring WebFlux en el backend, PostgreSQL para la base de datos, y RapidAPI para la inteligencia artificial."

### Minuto 2-4: CREATE
"Primero voy a crear tres artículos. Ingreso la URL, selecciono el idioma y la longitud del resumen. La aplicación llama a la API de IA y genera el resumen automáticamente."

### Minuto 4-6: READ
"Aquí vemos todos los artículos creados. Puedo filtrar por estado: pendientes, generados o fallidos. También puedo filtrar por idioma: español o inglés."

### Minuto 6-8: UPDATE
"Ahora voy a editar un artículo. Cambio la longitud del resumen y la aplicación regenera el resumen usando la IA nuevamente."

### Minuto 8-9: DELETE
"Para eliminar, hago clic en el botón rojo, confirmo la acción y el artículo se elimina permanentemente de la base de datos."

### Minuto 9-10: Base de Datos
"Aquí en PostgreSQL podemos ver todos los registros almacenados, con sus estados, resúmenes y fechas de creación."

### Minuto 10-11: Swagger
"El backend tiene documentación automática con Swagger UI, donde podemos ver todos los endpoints disponibles."

### Minuto 11-12: Conclusión
"Este proyecto demuestra un CRUD completo con integración de IA, arquitectura reactiva y diseño moderno. Gracias por ver."
